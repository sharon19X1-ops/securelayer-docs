⚠ Why Splunk, Defender, and Group-IB cannot cover this — and why that matters for A1

✗An AI agent accessing systems with stolen credentials generates no human login event, no geolocation anomaly, no Splunk rule match. The agent is a legitimate process — it just has someone else's token.
✗A prompt injection attack carries no email header, no URL, no file hash. Group-IB has no IOC to match. The malicious instruction arrives inside an API response — invisible to every SIEM.
✗An MCP server exfiltrating database records via tool calls looks like normal API traffic to a CASB. The agent is authorised. The volume looks plausible. No network anomaly fires.
These are live, documented attack vectors in 2025–2026 enterprise environments — not theoretical. Every SMB client A1 serves that runs AI coding tools has this gap today. SecureLayer is the only layer that closes it, and it routes every detection through the Torq workflow A1 already operates.
