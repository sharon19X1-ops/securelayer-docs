◎ Value Proposition · Splunk Edition

Splunk sees every threat
except the one growing fastest.

A1's Splunk deployment ingests logs from endpoints, identity, cloud, and network. It correlates. It alerts. It has no data source for AI agents — and no correlation rule can fire on data that never arrives. SecureLayer is the AI agent telemetry layer that Splunk is missing. Together, A1 delivers complete SIEM coverage for the first time.

The one-line value proposition for A1's Splunk clients:

SecureLayer is the EDR for AI agents — and it speaks Splunk natively. Every AI agent event SecureLayer detects fires into Splunk via HEC in Splunk CIM-normalized format. A1's existing correlation searches, Splunk ES notable events, and dashboard content apply immediately — with zero custom integration work. Splunk gains a new threat category. A1 gains a managed service built on infrastructure its clients already pay for.

Three reasons this makes A1's Splunk investment more valuable
01
Splunk gains a data source it cannot build itself
No Splunk add-on, no technology partner, and no existing endpoint agent generates AI agent telemetry. SecureLayer is the only source. Adding it to Splunk expands what A1's SIEM can see — permanently.
02
CIM-normalized — works with A1's existing content
SecureLayer events land in Splunk as CIM-normalized alerts: action, dest, severity, src_user, vendor. A1's existing Splunk ES correlation searches, risk-based alerting, and dashboards pick them up immediately. No new detection content to write.
03
Splunk alert → Torq → full response chain
SecureLayer fires into Splunk. Splunk fires a notable event. Torq picks it up and runs the response playbook — Teams approval, Entra ID action, MCP server isolation. All three A1 layers benefit from one new data source.

⚠ What Splunk is missing today — and why no existing add-on fixes it

✗An AI agent (Claude Code, Cursor, Copilot) authenticating with a stolen token — no login event fires, no Splunk auth log entry exists. The agent is a process, not a user session.
✗A prompt injection attack delivering malicious instructions to an agent mid-session — no network event, no file hash, no URL. Nothing Splunk can ingest from existing sources.
✗An MCP server exfiltrating database records via tool calls — traffic looks like normal API usage. No Splunk network rule fires. No anomaly in endpoint logs.
✗An AI agent running a shell-exec-sudo skill at 2am — the process is whitelisted. Splunk sees a legitimate executable doing what it was allowed to do. No alert.
These are not Splunk gaps — they are data source gaps. Splunk cannot correlate data it never receives. SecureLayer is the sensor that generates that data.

Native Splunk Integration — How SecureLayer Connects

How events reach Splunk

→HTTP Event Collector (HEC) — industry-standard Splunk ingest
→Splunk CIM-normalized fields — works with existing content packs
→Sourcetype: slb:alert — searchable and reportable immediately
→Index: security — lands in A1's existing security index
→Latency: <100ms from detection to Splunk ingest
→Retry + dead-letter queue — no missed events on Splunk downtime
